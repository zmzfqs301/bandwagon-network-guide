## 搬瓦工线路分几档，分别是什么

搬瓦工目前在售套餐里涉及的线路，从低到高大致分三档：

**第一档：163 骨干网（普通线路）**
对应 KVM 入门套餐，走中国电信普通国际出口，俗称 163 线路。白天够用，晚高峰（晚八点到十一点）容易拥堵，延迟飘，丢包率高。适合预算紧、主要用来学 Linux 或跑脚本的场景，不适合对速度有要求的建站或日常使用。

**第二档：CN2 GT（中端过渡线路）**
出境段走 CN2 专线，但进入国内后还是归并到 163 骨干网出口。好比高速公路开到省界口就换回了老国道——中间那段顺，最后那段看运气。晚高峰会有抖动，但整体比纯 163 强一截。现在搬瓦工已经基本砍掉了 CN2 GT 这个中间档，直接从普通线路跳到 CN2 GIA，DC3 机房早在 2025 年就下架了，不用再纠结这个选项。

**第三档：CN2 GIA / CMIN2 / CUP（高端优化线路）**
这才是搬瓦工最值得买的那一层。全程走专用通道，进出中国都不经过 163 拥堵节点。晚高峰延迟和丢包控制得很稳，这才是真正意义上的"有线路要求"。

## 三大运营商对应什么线路

这是实际选购时最关键的一个问题，搬瓦工线路要求和你的宽带运营商直接挂钩。

**电信用户**
对应线路：CN2 GIA。这是中国电信自己的精品骨干网，去程和回程都走 59.43 开头的 CN2 节点，全程不混入 163，晚高峰稳定性最好。DC6 和 DC9 两个机房的电信体验基本是当前美国节点里最稳的选择之一。

**联通用户**
对应线路：CUP（AS10099，国际段）接 AS9929（国内段）。联通以前用搬瓦工体验一般，2025 年 DC6/DC9 接入 CUP 线路之后，联通去程走 AS9929 精品网，体验有了明显提升。此外，荷兰 EUNL_9 机房也是联通走 CUP 线路，如果需要欧洲节点可以选这个。另外日本软银机房对联通也比较友好，延迟在 60-80ms 左右，比美国低很多。

**移动用户**
对应线路：CMIN2（China Mobile International Next-generation Network，AS58807）。移动用户以前是搬瓦工三大运营商里体验最差的——CN2 GIA 是电信的线路，移动用要绕路。DC6/DC9 接入 CMIN2 之后，移动用户终于有了自己的直连精品线路，实测延迟 150ms 左右，丢包率控制得很好。

简洁版对应关系：

|--------|----------|----------|
| 电信 | CN2 GIA | DC6 / DC9 / DC5 SLA |
| 联通 | CUP + AS9929 | DC6 / DC9 / 荷兰 EUNL_9 |
| 移动 | CMIN2 | DC6 / DC9 |

## 怎么判断自己走的是什么线路

买了 VPS 之后可以在机器上跑一次 `traceroute`（Windows 下叫 `tracert`）到国内 IP，看中间节点的 IP 前缀：

1. 路由节点带 **59.43** 开头 → CN2 节点，说明走了 CN2 线路
2. 去程和回程都带 59.43 → CN2 GIA（全程）
3. 只有出境段带 59.43，回程变 202.97 → CN2 GT（半程）
4. 节点全是 **202.97** 开头 → 163 骨干网（普通线路）

买 CN2 GIA-E 套餐之后，如果在 KiwiVM 切到 DC6 或 DC9 机房，用 traceroute 验证一下，正常应该全程都是 59.43 节点。如果发现走了普通线路，大概率是机房没切到位，重新迁移一次。
